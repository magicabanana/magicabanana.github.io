# Toolbox 프로젝트 구조 가이드 (AI 분석용)

이 문서는 AI 어시스턴트가 프로젝트 전반을 빠르고 정확하게 파악할 수 있도록 작성된 구조 분석 가이드입니다.

## 1. 개요
*   **프로젝트 디렉토리**: `c:\Myweb\toolbox\`
*   **목적**: 이미지 및 애니메이션 처리와 관련된 개별 웹 도구들을 한 곳에서 접근할 수 있도록 모아둔 **툴박스(허브) 형태의 정적 웹사이트**.

## 2. 주요 구성 요소

### 2.1 메인 홈 영역 `index.html`
*   **역할**: 허브 페이지의 뼈대
*   **특징**: 각 툴의 버튼을 마크업(HTML) 하드코딩으로 넣은 구조가 아닙니다.
*   본문 공간에 비어있는 `<div id="tools-grid"></div>` 컨테이너만 있으며, 자바스크립트를 이용해 연결된 도구 목록의 메뉴(카드 UI)를 동적으로 채워 넣습니다.
*   언어 전환(한국어/영어) 네비게이션이 포함되어 있습니다.

### 2.2 설정 파일 `config/tools.js`
*   **역할**: 전시될 애플리케이션(툴)들의 구성 정보(메타데이터) 보관소
*   **구조**: `TOOLBOX_CONFIG` 객체 내부의 배열 요소에 등록됩니다.
*   **키값**:
    *   `id`: 도구의 고유 식별자
    *   `title` / `title_en`: 한국어 및 영어 도구 이름
    *   `description` / `description_en`: 도구에 대한 간단한 설명
    *   **`path`**: 해당 툴의 실제 폴더 및 웹페이지 경로(`tools/{도구이름}/index.html`)
    *   `icon`: 카드 렌더링에 사용되는 이모지 혹은 아이콘

### 2.3 핵심 제어 스크립트 `script.js`
*   **동적 UI 렌더링**: `config/tools.js`에서 불러온 데이터를 사용하여 `tools-grid` 안에 `<a class="card" href="{path}">...</a>` 형식의 링크 카드를 만들어냅니다.
*   **배경 애니메이션 효과**: 캔버스(`canvas`)를 활용하여 마우스와 무관하게 배경에 표류하는 파티클(입자)과 선 형태의 비주얼 이펙트를 렌더링하여 역동성을 줍니다.
*   **다국어 지원**: `lang-toggle` 버튼 클릭 시 `data-ko`와 `data-en` 속성을 번갈아 스위칭하여 메뉴 언어를 변경합니다.

### 2.4 도구 보관소 `tools/`
*   **역할**: 허브에서 클릭했을 때 이동되는 실제 각각의 개별 웹 애플리케이션 코드가 위치한 저장소 폴더
*   **내부 구조**: 각 도구별로 독립적인 폴더(예: `다중PNG시트변환기`, `이미지압축기`, `워터마크생성기`)가 나뉘어 있으며, 각 폴더 안에는 본인만의 로직(HTML, CSS, JS)이 들어있습니다.

## 3. 요약 (확장 프로세스)
이 툴박스에 **새로운 툴을 추가하려면** 아래 2단계만 거치면 됩니다.
1. `tools/` 이하에 구현한 툴의 새로운 폴더 및 파일을 복사·생성.
2. `config/tools.js` 배열 안에 새 항목으로 이름, 설명, 방금 생성한 경로 등을 추가. (그러면 `script.js`가 알아서 메인 화면에 카드를 생성함)
